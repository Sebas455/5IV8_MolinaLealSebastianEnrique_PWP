Todos los forms en la web, manejan métodos para le envío de la dif peticiones que realiza el usuario considerando la arquitectura cliente-servidor el manejo de las peticiones s ehace por medio del protocolo http y actualmente por htpps, cada petición debe de ser atendida en un tiempo de respuesta conforme al modelo TCP/IP. Lod métodos principales de los formularios son:

    • Get:

      Este método se encarga de enviar la info del formulario por medio de la salida estandar, (conocida como URL), del navegador, esto propicia que puedan existir inyecciones de código realizando violaciones a la seguridad del sistema

      POST:

      Este método se encarga de enviar la info para poder registrar un cambio o un nuevo registro dentro del sistema enviandola igual por el método estandar; pero mediante un ocultamiento, esto significa que no es visible para el usuario lo que nos ofrece un mayor grado de seguridad en el manejo de la información


    • En resumen los métodos son:

    • Get: Sirve para enviar y realizar operaciones de lectura o consulta
    • Post: Sirve para enviar info de un recurso al servidor, con el fin de registrarlo
    • Delete: Sirve para eliminar un recurso específico dentro del servidor obteniendo una respuesta de actualización
    • Put: Sirve para almacenar la info de un recurso particular y conectarse al servicio paravrealizar la actyalización
    • Head: Sirve para regresar el codigo de puesta de cualquier cabecera HTTP asociada a un recurso
